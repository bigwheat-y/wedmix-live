# WedMix Live

婚礼现场专业调音控制台 -- 面向婚礼调音师设计的 Web 实时音频混音工具。

## 简介

WedMix Live 是一款纯前端、零服务器依赖的婚礼现场音频控制台。基于 Web Audio API 构建，提供仪式流程流控、气氛打击垫、交叉淡化、压限避让与紧急静音等专业级功能，适用于婚礼现场的 BGM 管理与实时音效触发。

## 功能特性

### 仪式流程流控
- 预设 8 个婚礼仪式环节（宾客暖场、新郎入场、新娘入场、誓言交换、信物交换、礼成退场、宴会敬酒等）
- 支持自定义添加、删除仪式环节
- 每个环节可独立配置淡入/淡出时间、起始/结束时间点、循环播放
- 一键触发场景切换，自动交叉淡化过渡

### 实时播放引擎
- 波形可视化窗口，实时显示播放进度
- 支持拖拽或点击上传音频文件（MP3/WAV 等浏览器支持的格式）
- 播放/暂停/停止/循环控制
- 音频文件持久化缓存至 IndexedDB，刷新页面后自动恢复

### 气氛打击垫 (Soundboard)
- 12 个即时触发音效垫，支持多声部叠加播放
- 内置 4 种高保真合成音效：教堂钟声、浪漫音垫、庆典礼乐、悬念扫频
- 支持切换为自定义音频文件模式
- 合成音效 100% 离线可用，无需网络加载

### 主控台
- 主音量推子 + 实时 LED VU 表（L/R 双声道）
- 手动交叉淡化控制器
- 司仪人声避让 (BGM Ducking) -- 一键压限背景音乐音量 75%
- 紧急一键淡出 (1.5 秒归零)
- 调音台面板锁定（防误触）

### 配置管理
- 自动保存当前配置至 localStorage
- 导出为 ZIP 包（含 config.json + 所有音频文件）
- 支持导入 ZIP 包或 JSON 配置文件
- 一键重置所有配置与缓存
- 内置演示音乐生成器（基于 OfflineAudioContext 合成）

## 技术栈

- **HTML / CSS / JavaScript** -- 纯前端，无框架依赖
- **Electron** -- 跨平台桌面应用封装（macOS / Windows）
- **Web Audio API** -- 音频播放、合成、分析
- **IndexedDB** -- 音频文件持久化缓存
- **JSZip** -- ZIP 配置包导出/导入（本地离线版本）
- **Canvas API** -- 波形可视化

## 产品详细文档

### 一、产品概述

**WedMix Live** 是一款专为婚礼现场调音师设计的 Web 端实时音频混音控制台。

**核心定位：**
- 面向婚礼司仪团队、现场调音师、婚庆策划师
- 解决婚礼现场 BGM 切换混乱、音效触发不及时、多环节衔接不顺畅的痛点
- 提供仪式流程标准化管理 + 实时音频控制的一体化方案

**产品形态：**
- 纯浏览器端应用，打开 HTML 文件即可使用
- 无需安装软件、无需注册账号、无需网络服务器
- 数据全部存储在本地浏览器（IndexedDB + localStorage），隐私安全

**核心价值：**
- 将婚礼现场的音频控制流程标准化、可视化
- 降低调音师的操作门槛，减少现场失误
- 支持离线运行，适应婚礼现场网络不稳定的环境

### 二、详细使用流程

#### 第一步：打开控制台

双击桌面的 **WedMix Live** 图标启动应用（或在开发模式下运行 `npm start`）。首次打开时，系统会显示一个加载动画（SYNCHRONIZING AUDIO CACHES），等待 IndexedDB 数据库初始化完成。如果之前没有缓存数据，会直接进入空白控制台。

#### 第二步：输入婚礼方案名称

在顶部状态栏的输入框中，输入本场婚礼的新人名称，例如"张先生 & 李女士婚礼配乐方案"。该名称会自动保存，并在导出配置时作为文件名的一部分。

#### 第三步：为仪式环节配乐

左侧"婚礼仪式主干流程流控"面板预设了 8 个标准婚礼环节：

| 序号 | 环节名称 | 默认淡入 | 默认淡出 |
|------|----------|----------|----------|
| 01 | 宾客进场暖场 | 2.0s | 2.0s |
| 02 | 司仪开场致辞 | 1.0s | 1.5s |
| 03 | 新郎帅气入场 | 1.5s | 2.0s |
| 04 | 新娘圣洁入场 | 2.0s | 2.5s |
| 05 | 誓言交换与致辞 | 2.5s | 2.0s |
| 06 | 信物交换与拥吻 | 1.0s | 2.0s |
| 07 | 礼成退场欢庆 | 1.0s | 2.5s |
| 08 | 宴会背景与敬酒 | 2.5s | 2.5s |

**配乐方式（二选一）：**
- **点击上传：** 点击每个环节下方的"📂 点击或拖拽音频文件配乐"区域，从文件管理器中选择音频文件
- **拖拽上传：** 直接将音频文件从文件管理器拖拽到对应环节的配乐槽中

上传后，系统会自动解码音频并缓存到 IndexedDB。已配乐的环节会显示文件名和时长，例如"🎵 婚礼进行曲.mp3 (3:45.2)"。

#### 第四步：配置环节参数

每个仪式环节支持以下可调参数：

- **环节名称：** 点击名称可直接编辑，用于标识当前环节
- **淡入时间 (Fade In)：** 该环节触发时，音量从 0 到满幅的过渡时长（0-10 秒，步进 0.5s）
- **淡出时间 (Fade Out)：** 切换到下一环节时，当前环节音量归零的过渡时长
- **起始时间 (Start)：** 音频从第几秒开始播放（跳过前奏等）
- **结束时间 (End)：** 音频播放到第几秒停止（留空表示播放到末尾）
- **循环播放 (Loop)：** 勾选后，该环节会在起始到结束时间范围内循环播放

所有参数修改后自动保存到浏览器本地存储。

#### 第五步：现场触发场景切换

在婚礼进行过程中，按照仪式进度依次点击对应环节的 [⚡ 触发] 按钮：

1. 点击后，当前正在播放的环节会按照其"淡出时间"自动降低音量
2. 新环节按照其"淡入时间"自动升高音量
3. 两个环节的音频会短暂重叠，实现无缝交叉淡化过渡
4. 当前活跃环节的状态标签会变为"ON AIR"，卡片高亮显示

**注意事项：**
- 只有已配乐的环节才能触发
- 同一环节重复触发不会重新开始
- 触发时会自动更新右侧波形窗口的显示

#### 第六步：使用气氛打击垫

在仪式进行中，可以随时点击右下方的气氛打击垫按钮，即时叠加播放音效：

- 音效会在当前 BGM 之上叠加播放，不会中断背景音乐
- 点击后按钮会有短暂的视觉反馈动画
- 默认使用合成音效，100% 离线可用

**切换为自定义音效：**
1. 点击打击垫右下角的齿轮图标 ⚙️
2. 在弹出的配置窗口中选择"自定义音频文件"
3. 上传自己的音效文件
4. 点击"💾 保存配置"

#### 第七步：使用主控台功能

底部主控台区域提供全局音频控制：

- **🚨 EMERGENCY FADE OUT：** 紧急情况下一键在 1.5 秒内将所有音频淡出至静音，并自动锁定面板
- **🎙️ 司仪人声避让：** 开启后，背景音乐音量自动压限至 25%，方便司仪讲话
- **交叉淡化控制器：** 手动调节 Deck A/B 之间的音量平衡
- **主音量推子：** 控制整体输出音量
- **VU 表：** 实时显示左右声道的输出电平

#### 第八步：锁定面板

在现场正式使用时，建议勾选"🔒 锁定调音台面板"复选框。锁定后，所有场景触发和播放控制按钮将被禁用，防止调音师在操作其他功能时误触导致音乐中断。

#### 第九步：保存与导出配置

婚礼结束后或配置完成后：

- **💾 保存配置：** 将当前方案导出为 ZIP 压缩包，包含 `config.json`（所有环节配置、打击垫设置）和 `audio/` 目录（所有已上传的音频文件）
- **📂 导入配置：** 在另一台设备上导入 ZIP 包或 JSON 配置文件，恢复完整的婚礼配乐方案
- **🔄 一键重置：** 清除所有配置和缓存数据，恢复到出厂默认状态（需要二次确认）

### 四、配置管理与数据存储

#### 4.1 存储架构

WedMix Live 使用三层本地存储机制：

**第一层：localStorage（配置数据）**
- 存储键名：`wedmix_current_config`
- 内容：JSON 格式的完整配置对象，包含方案名称、所有环节参数、打击垫设置
- 特点：每次参数变更时自动保存，页面刷新后自动恢复配置结构
- 限制：不存储音频文件本身（localStorage 容量有限）

**第二层：IndexedDB（音频文件缓存）**
- 数据库名：`WedMixLiveDB`
- 对象仓库：`tracks`（存储音频 Blob）、`configs`（存储配置档案）
- 缓存键名规则：
  - 仪式环节：`scene_{环节ID}_active`
  - 打击垫：`pad_{垫ID}_active`
- 特点：大容量持久化存储，音频文件在页面刷新后自动从 IndexedDB 恢复并重新解码

**第三层：ZIP 导出包（离线备份）**
- 文件结构：
  ```
  WedMix_{方案名称}_{日期}.zip
  ├── config.json                    # 完整配置
  └── audio/
      ├── scene_1_宾客暖场.mp3       # 环节 1 的音频
      ├── scene_2_司仪开场.mp3       # 环节 2 的音频
      ├── ...
      ├── pad_3_自定义音效.wav       # 打击垫 3 的自定义音频
      └── ...
  ```
- 特点：可跨设备传输，完整备份所有配置和音频

#### 4.2 配置文件格式 (config.json)

```json
{
  "version": "2.0",
  "exportedAt": "2026-05-28T10:00:00.000Z",
  "schemeName": "张先生 & 李女士婚礼配乐方案",
  "crossfade": 2.5,
  "scenes": [
    {
      "id": 1,
      "name": "宾客进场暖场",
      "assignedTrackId": "scene_1_active",
      "assignedTrackName": "暖场音乐.mp3",
      "audioZipPath": "audio/scene_1_暖场音乐.mp3",
      "fadeIn": 2.0,
      "fadeOut": 2.0,
      "startTime": 0,
      "endTime": null,
      "loop": false
    }
  ],
  "soundboard": [
    {
      "id": 1,
      "name": "掌声雷动",
      "type": "synth",
      "synthEffectId": "fanfare",
      "assignedTrackId": null,
      "assignedTrackName": null,
      "audioZipPath": null
    }
  ]
}
```

#### 4.3 数据生命周期

1. **首次使用：** 控制台加载默认配置（8 个标准环节 + 12 个默认打击垫）
2. **配乐上传：** 音频文件同时写入 IndexedDB 和解码到内存，配置变更写入 localStorage
3. **页面刷新：** 先从 localStorage 恢复配置结构，再从 IndexedDB 恢复音频文件并重新解码
4. **导出配置：** 从 IndexedDB 读取音频 Blob，连同配置 JSON 一起打包为 ZIP
5. **导入配置：** 解析 ZIP 中的 config.json 恢复配置，将音频文件写入 IndexedDB 并解码
6. **一键重置：** 清空 IndexedDB 所有缓存 + 删除 localStorage 配置 + 恢复内存默认值

#### 4.4 自动保存机制

以下操作会触发自动保存到 localStorage：
- 修改婚礼方案名称
- 编辑环节名称
- 修改淡入/淡出/起始/结束时间
- 切换循环开关
- 上传音频文件到环节
- 保存打击垫配置
- 新增或删除仪式环节
- 导入配置文件

自动保存是静默执行的，无需手动操作。浏览器控制台会输出"自动保存当前页面配置到 localStorage"的日志。

#### 4.5 跨设备使用流程

**场景：在办公室电脑上配置好音乐，带到婚礼现场使用**

1. 在办公室电脑上完成所有环节配乐和参数调整
2. 点击"💾 保存配置"，下载 ZIP 配置包
3. 将 ZIP 文件通过 U 盘或即时通讯工具传输到现场电脑
4. 在现场电脑上打开 WedMix Live，点击"📂 导入配置"
5. 选择 ZIP 文件，等待导入完成
6. 所有配置和音频自动恢复，即可开始使用
